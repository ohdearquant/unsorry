Added an `auto-archive` workflow (hourly + manual `workflow_dispatch`) that keeps the active proved set bounded automatically (ADR-041 / SPEC-041-A §9). When active proved-not-archived crosses a ceiling (default 40) and no archive PR is already open, `tools/archive/auto_cut.sh` cuts one block with the write-mode tool, validates it (Gate B on the active tree and the package, ADR-018 goal immutability, zero generated-doc delta), and opens an auto-merge retire PR — serialising one block per merge so it never floods Gate A. This is the durable fix for the **exit-137 OOM**: a growing active set makes Gate A's full kernel replay and `lake build UnsorryLibrary --wfail` exceed runner memory; capping the active set keeps both under memory (archived proofs are provenance+packaging validated, not re-replayed, ADR-048). Needs the dispatcher's `REFRESH_TOKEN` (a GITHUB_TOKEN-opened PR doesn't trigger Gate A).
