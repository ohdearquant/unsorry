**Model → Pokémon registry & the first swarm operational task** (ADR-083). A new `swarm/housekeeping.sh` work package — run first by `run.sh` — researches each model in the leaderboard distribution (open/closed source, publisher, country, parameter size, canonical link) and assigns it a unique Pokémon identity, published to `docs/metrics/model-registry.json` for the guild frontend. One Pokémon per model per PR, enforced by a deterministic validator (`tools/model_registry`, stdlib + pytest) and the `model-registry-gate` workflow (schema · Pokémon validity · uniqueness · one-new-entry diff). Seeded for the 11 models currently on the board.
