Gate A now mounts a persistent Namespace `.lake` cache volume (`nscloud-cache-action`, ADR-046) at `${GITHUB_WORKSPACE}/.lake` in the prepare/audit/replay jobs, so mathlib oleans and the local build survive across jobs and runs — eliminating the ~2–3.5-min GitHub-cache mathlib restore paid on every job, the last constant cost after ADR-045. Failsafe: the volume engages only on a `namespace-*` runner profile (a `detect.volume` flag) and the mount is `continue-on-error`, so non-Namespace runners (or a profile with no volume attached) fall back to the existing GitHub mathlib + `.lake/build` caches with no behavioural change. `use-github-cache` is turned off only when the volume is in play. Soundness unchanged — the cache is advisory; `leanchecker` re-checks every olean against the kernel and the push-to-`main` full replay backstops. Operator step: attach a cache volume (~20 GB) to both runner profiles.
