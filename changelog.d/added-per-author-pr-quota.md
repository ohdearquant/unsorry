Added a per-contributor open-PR fairness cap (ADR-054 quota layer), enforced in CI by `pr-admission.yml`. The shared open-prove-PR budget (`UNSORRY_MAX_OPEN_PROVE_PRS`) is global, so without a per-author bound a single contributor's fleet can fill it and pause submissions for everyone — the "CI flooding / credit gaming" ADR-054 names, observed in practice as one author holding all 43 open prove PRs. On `opened`/`reopened` of a `queued/prove/*` PR, admission now counts the author's open prove PRs and closes the newest over-cap submission (FIFO: a contributor's oldest PRs keep draining; nothing already admitted is touched). The cap defaults to 20 and is tunable without a code change via the `UNSORRY_MAX_OPEN_PROVE_PRS_PER_AUTHOR` repo variable. `tools.repo.pr_admission` gains a pure, tested `quota_decide()` and a `quota` CLI subcommand.
