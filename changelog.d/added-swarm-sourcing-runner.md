Added `swarm/sourcing.sh`, the goal-sourcing runner ([ADR-062](docs/adrs/ADR-062-Swarm-Goal-Sourcing-Runner.md) / [SPEC-062-A](docs/adrs/specs/SPEC-062-A-Swarm-Goal-Sourcing-Runner.md)) — the sourcing counterpart to `swarm/agent.sh`. It fires up Claude to run one cycle of the `unsorry-goal-sourcing` skill (ADR-060): pick a hard theme, run the four sourcing gates over candidates, promote survivors to triples with `gen_triples.py`, and open one `chore(sourcing):` PR of `≤ --max-goals` goals. Unlike the prove loop it is **bounded by default** (one cycle; `--cycles N` / `UNSORRY_SOURCING_CYCLES` to repeat) because sourcing has no empty-pool terminator, supports `--theme`, `--max-goals` (clamped to 1..50), and `--dry-run`, and reuses `agent.sh`'s preflight, ADR-016 infrastructure-failure classification, ADR-059 fetch resilience, and 0/1/2/3 exit codes so `swarm/supervise.sh` wraps it unchanged. Driven by a new `swarm/prompts/source.md` playbook; shellcheck-clean with a hermetic `--self-test`, both wired into `agent-lint.yml`.
