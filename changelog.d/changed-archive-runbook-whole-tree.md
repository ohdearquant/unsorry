Corrected the archive-cut runbook (SPEC-041-A §8/§9) after blocks 0003/0004 surfaced two invariants the original runbook got wrong: **(A) archive whole decomposition trees only** — a decomposition record + its parent + subs are atomic to Gate B (GB008 src-exists / GB016 parent-known), so a tree must move to the package entirely or not at all (the planner's flat top-N splits trees, so the cut must restrict to whole trees + standalone goals, usually <40); and **(B) never regenerate generated docs in the cut** — `docs/leaderboard.*`, `docs/metrics/*.json`, `docs/targets.md`, and the proof-graph/visualisation outputs are refreshed by push-to-`main` workflows (no PR gate), so regenerating them in the PR races main's refresh bot and conflicts perpetually; leave them at the merge-base. Added the package-tree validation step (`gate_b validate <pkg> --goals-root <pkg>`) and the zero-doc-delta check. The §9 automation note now requires the future `tools.archive` write mode to encode both invariants.
