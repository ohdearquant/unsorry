Added a write-mode archive cut (`python3 -m tools.archive.apply`, ADR-041 / SPEC-041-A §9) and used it to archive 120 proofs into blocks **0005–0007** (3×40). The §8 cut runbook was previously hand-only and the spec flagged it as where blocks 0003/0004 had failed validation; the tool bakes in both hard invariants — (A) it archives whole decomposition trees only (tree-aware selection over the active decomposition graph, never splitting one) and (B) it writes nothing under `docs/` (generated boards are left to the push-to-`main` refresh workflows). It moves each proof's module, `library/index` entry, `goals/<id>.lean`, `backlog/<id>.md`, and `proof-runs`; copies `goals/<id>.aisp` into the package; re-points each active goal record to the archived end-state (`status≜archived`, paths prefixed, `sha` unchanged); moves whole-tree decomposition records; and writes the package `lakefile.toml`/`lean-toolchain`/`lake-manifest.json`/`archive-manifest.json`. Seed/translate goals with `lean≜∅` (e.g. `Unsorry.Basic`) are skipped. Per ADR-048, archived proofs are validated by provenance + packaging, not re-replayed — so shrinking the active set keeps Gate A's full-replay long pole fast as proof inflow grows.
