Gate A's `gate-a-prepare` and `gate-a-audit` jobs now allow 45 minutes (was 30); `gate-a-replay` stays at 60. A cold library build (~21 min and growing with the active module count) plus the per-job checks overran the old 30-minute cap and the job was killed, blocking PRs whose run had no warm `.lake/build` cache to restore (#567/#573). The ADR-045 build cache keeps the warm path to ~minutes; this only adds headroom for the rare cold path (first run on a new cache prefix, e.g. after a toolchain/mathlib bump, or before a push-to-`main` run has saved a main-scoped cache). The durable bound on cold-build time remains ADR-041 archiving (keeping the active library small), not a larger timeout. Documented in SPEC-045-A.
