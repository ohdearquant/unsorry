Gate A now caches the local Lake build directory (`.lake/build`) across runs, so `lake build UnsorryLibrary` is incremental instead of cold-building the whole library in every job (ADR-045/SPEC-045-A). Previously each of the three Lean jobs (`gate-a-prepare`, `gate-a-audit`, `gate-a-replay`) recompiled the entire library from scratch — measured at ~16-21 min **per job** on a one-proof PR — even though mathlib was already cached and the audit/replay are incremental. An `actions/cache` step (keyed on toolchain/mathlib/lakefile + commit sha, with a restore-keys prefix) now restores the most recent oleans so only changed modules recompile; within a run `gate-a-prepare` builds once and `gate-a-audit`/`gate-a-replay` restore that exact build, so the library is built once per run instead of three times, and re-runs of a cancelled run skip the build entirely. Soundness is unchanged: Lake's content-hash traces recompile any changed source, and the axiom audit + `leanchecker` kernel replay still validate every olean that reaches `main` (full replay on push to `main`), cached or not. The cache rides the GitHub Actions cache protocol that the Namespace runners already back with high-performance storage.
