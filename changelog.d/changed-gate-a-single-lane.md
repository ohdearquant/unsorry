Consolidated all Gate A Namespace stages (prepare, audit, replay, archive) onto a single runner lane `namespace-profile-unsorry-1`, replacing the per-stage prepare/audit/replay lanes (ADR-058). Maintaining three separate lanes by hand repeatedly left one lane at zero runners — and a stage whose lane has no runner queues forever, hanging the entire Gate A run and stalling every merge (observed: hours with zero proofs merging while the prepare or replay lane sat empty). With one shared pool, any Namespace runner serves any stage regardless of the workload mix, so the operator maintains exactly one label and there is no lane to leave empty. The scheduled full-replay backstop moves to the same lane. Trade-off: replay (the memory-heavy `leanchecker` stage) now shares the pool, so the `unsorry-1` runners must be sized for it (≥8 GB); `UNSORRY_REPLAY_CHUNK` and auto-archive (smaller active set) keep per-run memory bounded.
