Narrowed the Gate A **full-replay** trigger (ADR-033) to **olean-invalidating changes only** (`lean-toolchain`, `lakefile.toml`, `lakefile.lean`, `lake-manifest.json`). A change to `tools/gate_a/**` or `.github/workflows/gate-a.yml` no longer forces a full kernel replay — those drive/schedule `leanchecker` but never change an olean, so they now run an **incremental** replay (a no-op when no library module changed). This stops gate/CI PRs from riding the memory-bound serial full replay that OOM-killed the 16 GB runner (exit 137, ADR-047). Soundness is preserved by the orchestration's unit tests, the push-to-`main` full-replay backstop, and the **axiom audit**, which keeps the conservative trigger (`tools/gate_a/**`, `AxiomAudit/`, `AuditFixtures/`, `gate-a.yml` still force a full audit — `collectAxioms` is light + parallel, so it does not OOM).
