Gate A no longer rebuilds `UnsorryLibrary` in the audit and replay jobs when the `.lake/build` cache hits on the exact commit-sha key (ADR-045 amendment). Run logs showed the cache restored but `lake build` re-elaborated the heavy library modules anyway (~3.5 min) in **all three** jobs, because `lean-action` re-provisions mathlib after the restore and Lake stops trusting the restored oleans. `gate-a-prepare` stays the authoritative `--wfail` builder; `gate-a-audit`/`gate-a-replay` now guard their build with `if: steps.lake_build_cache.outputs.cache-hit != 'true'` and simply **load** prepare's exact-commit oleans (`axiom_audit` via `lake exe`, `leanchecker` via `lake env` — neither rebuilds the library). Self-healing on a cache miss; soundness unchanged (the kernel replay still checks every olean it loads, and the push-to-`main` full replay backstops). Removes the gate's biggest constant cost — the audit job drops from ~535 s to ~320 s.
