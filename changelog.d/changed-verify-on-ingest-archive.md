Adopted **verify-on-ingest** for archive validation (ADR-048 + SPEC-048-A): an archive-block PR validates **provenance + packaging** — each archived proof module is proven **byte-identical** (git-blob identity) to the already-verified version via the new `archive_proof_provenance` check (base active module or base archived copy; net-new or altered proof content is rejected), goal statements stay pinned by ADR-018, and `lake build --wfail` keeps packaging sanity — and it **no longer re-runs `leanchecker` kernel replay** on archived proofs. Those proofs were kernel-verified by Gate A when active and are byte-identical now, so re-replay re-proves nothing; it also loaded each archive package's full mathlib image and OOM-killed even a 16 GB runner (#764). Removes the archive-OOM class (archive PRs fit the 8 GB runner) and supersedes the tactical chunking (#823) and 16 GB-pin (#838). Lean soundness is unchanged — each proof is still kernel-replayed once, at its introducing PR, under its toolchain/mathlib context, and a context change (toolchain/mathlib/lake/checker) still forces re-verification. The residual risk is bookkeeping (the archived file must be exactly the verified one), closed by the git-blob provenance check. Phase 2 (push-to-main full replay → scheduled/incremental + downscale CI 16 GB → 8 GB) is a documented follow-up.
