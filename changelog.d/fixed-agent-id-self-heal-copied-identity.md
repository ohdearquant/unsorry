Made swarm identity correct-by-default so a copied setup can no longer silently steal proof credit. The agent-id default already encodes the machine's hostname (`<host>-<4 hex>`), so a persisted `~/.unsorry/agent-id` whose host-prefix doesn't match the local machine was demonstrably generated elsewhere (a cloned setup). `swarm/agent.sh:resolve_agent_id` now detects that case and **auto-regenerates a fresh local id** — no manual `UNSORRY_AGENT_ID` needed. This is what let multiple contributors run under one shared `mac-158f` identity (mis-crediting proofs and colliding on claims). An explicitly exported `UNSORRY_AGENT_ID` and custom-shaped ids (suffix not 4 hex) are honoured untouched. Together with the `run.sh` solver-credit guard, cloning a repo now yields *your own* identity automatically: solver defaults to your `gh` login, agent-id to your host. New pure helpers `local_host` / `agent_id_host_matches` are unit-tested in `agent.sh --self-test`.
