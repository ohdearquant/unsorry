Fixed the archive-package validation OOM-killing replay (exit 137) on memory-bound runners. `tools/gate_a/archive_packages.py` ran `leanchecker` over **every** module of a frozen block in a single invocation; since `leanchecker` holds ~all of mathlib resident (~7 GB), a ~30-proof block (≈60 modules with bindings) exceeded a 16 GB runner and was OOM-killed — blocking archive-block PRs (#764). Replay now **chunks at `REPLAY_CHUNK_SIZE`** and runs the chunks serially, exactly like the active-library kernel replay, bounding peak memory. No soundness change: every module is still kernel-replayed, just in memory-safe batches. Regression test added.
