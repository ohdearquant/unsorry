Fixed `gate-a-archive` exhausting runner disk on wide metadata PRs (e.g. #3218's 1831-file attribution correction). `validate-changed` rebuilt every changed archive package — each running `lake exe cache get` (a multi-GB mathlib fetch/clone) — so a PR touching index records across all 32 packages ran out of disk (`No space left on device`). For a change confined to `library/index/*.aisp` (attribution/provenance metadata), the proof modules, lakefile, and toolchain are byte-identical — the ADR-048 provenance guard already proves it, and the `lake build` only re-checks packaging that a metadata edit cannot break. The validator now fast-paths such changes (provenance + Gate B metadata + binding generation still run; the redundant rebuild is skipped), which also speeds up the hourly auto-archive (it only ever adds index records). New pure predicate `only_index_metadata_changed` is unit-tested.
