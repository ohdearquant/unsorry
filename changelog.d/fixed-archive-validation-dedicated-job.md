Archive-package validation (ADR-041) now runs in its own `gate-a-archive` job instead of inside `gate-a-prepare`, fixing the exit-137 OOM that blocked block-retirement PRs (#764). Each archive package is a separate Lake project whose `leanchecker` replay loads its own full mathlib image (~7–10 GB); running that in `gate-a-prepare` (which already holds the freshly-built active library) exceeded the 16 GB runner even after #783's chunking. The dedicated job gives it full headroom + swap + the 120-min budget the active replay uses, runs in parallel (`needs: [detect]`, `if: archive == 'true'`), mounts the Namespace volume so each package's `lake exe cache get` reuses the shared mathlib `.ltar` cache (ADR-046), and `archive_packages` chunks the replay at the smaller `ARCHIVE_REPLAY_CHUNK_SIZE` (default 12, override `UNSORRY_ARCHIVE_REPLAY_CHUNK`). Soundness unchanged — the archived proofs are still `lake build --wfail`-compiled and kernel-replayed; the change is purely where and how the replay is scheduled.
