The proof-credit guard now runs on the universal prover chokepoint (`swarm/agent.sh:resolve_solver` setup), not only in `swarm/run.sh`. The guard previously lived only in `run.sh`, but the documented way to add a prover to an existing deployment is `./swarm/supervise.sh --prove` (and `./swarm/agent.sh` can be run directly) — both bypass `run.sh`, so a hard-coded `UNSORRY_SOLVER` credited someone else for every proof with no check. `agent.sh` now compares the resolved solver to the authenticated GitHub login before any proving and exits 2 (config-fatal, no retry) on a silent mismatch, warning instead when it's an acknowledged delegation (`UNSORRY_SOLVER_OK=1`) or the login is unresolvable (offline). Because the check sits at `resolve_solver` and re-runs on every #428 harness re-exec, a long-lived loop enforces it as soon as it picks up the latest harness — closing the gap that let multiple contributors funnel credit to a shared `UNSORRY_SOLVER=ohdearquant` config. New pure `solver_credit_decision` helper, unit-tested in `agent.sh --self-test`.
