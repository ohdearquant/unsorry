Fixed duplicate "already proved" prove PRs that kept re-appearing after ADR-064's dispatch dedup shipped ([ADR-071](docs/adrs/ADR-071-Fresh-Dispatch-Dedup-Recheck.md) / [SPEC-071-A](docs/adrs/specs/SPEC-071-A-Fresh-Dispatch-Dedup-Recheck.md)). ADR-064 evaluated its dedup checks from a snapshot taken once at the start of a dispatch pass; during the pass — and in the gap before `gh pr create` — a sibling proof of the same goal could merge (or a concurrent dispatcher open a PR), so the branch was dispatched into a dead duplicate that first-merge-wins then stranded as un-mergeable. The dispatcher now runs a fresh re-check (`goal_taken_fresh`: re-fetch `origin/main` + re-list open prove PRs) immediately before each create, for only the handful actually being dispatched — using `git grep` + one core-API list, never the 30/min search API. This shrinks the duplicate window from minutes-per-pass to a millisecond TOCTOU (kept sound by first-merge-wins). The upstream prove-time race that produces sibling branches, and a fully atomic dispatch lock, remain tracked follow-ups.
