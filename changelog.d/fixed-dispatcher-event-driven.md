Made the queue-dispatcher event-driven so the PR queue no longer sits empty while ready proof branches wait. GitHub throttles `schedule` triggers under load, so the `*/15` cron actually ran 1–5h apart exactly when the backlog was deepest — between runs nothing turned `queued/prove/*` branches into PRs even with hundreds ready. The dispatcher now also triggers on `workflow_run` (Gate A completion): a completion frees an in-flight slot, which is precisely when the governor-capped dispatcher can admit more, and completions aren't cron-throttled. The cron remains a low-activity backstop; the existing concurrency group still serialises passes so bursts never double-count in-flight work.
