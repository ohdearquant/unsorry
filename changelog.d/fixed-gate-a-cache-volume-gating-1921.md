Fixed the Gate A GitHub `.lake/build` cache running on **every** runner profile, including Namespace, because three `if:` guards referenced a non-existent `needs.detect.outputs.volume` output instead of the per-job `prepare_volume`/`audit_volume`/`replay_volume` (#1921). `'' != 'true'` is always true, so on Namespace profiles — where the persistent `.lake` volume (ADR-046) already keeps oleans warm — a ~320 MB per-commit cache entry was written every job, churning and **evicting the warm entries** that prevent cold builds. The guards now match the `use-github-cache` lines, so the GitHub cache runs only on non-Namespace profiles as ADR-046 intended. Also added a `::warning::` to each Namespace cache-diagnostics step when the volume misses, so a cold-build-causing miss is visible to operators rather than silent. (The structural cold-start levers — automating ADR-041 archiving to bound the active library, and a nightly warm-seed — remain scoped follow-ups in #1921.)
