Gate A's goal-immutability check (`tools/gate_a/check_goal_immutability.py`, ADR-018) is now **archive-aware**, unblocking ADR-041 archive-block retirement. Previously any `D goals/<id>.lean` was rejected unconditionally, so a PR retiring a frozen archive block (which deletes the now-redundant active goal copies) failed `gate-a-prepare` instantly. The check now exempts a delete **iff** the goal id is recorded in an archive manifest (`packages/unsorry-archive-*/archive-manifest.json`) **and** the archived `packages/<block>/goals/<id>.lean` is byte-identical to the deleted statement at the base ref — the pin relocates, unchanged, into a frozen and separately-validated archive block rather than vanishing. Modify/rename/typechange, a delete with no manifest entry, and a delete whose archived statement differs from history all stay rejected (the base-ref comparison is the tampering guard). ADR-018 amended, SPEC-018-A updated; new pure-core (`archive_retired`) and CLI tests.
