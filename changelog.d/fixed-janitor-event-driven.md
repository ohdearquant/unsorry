Fixed the stale-run janitor firing hours apart instead of every 20 min: GitHub throttles `schedule` triggers under load — exactly when the Gate A backlog (and the rate of zombie runs from dead Namespace runners) is highest — so cron-only scheduling let zombies pile up and re-clog the in-flight budget between sweeps. The janitor now triggers on `workflow_run` (every Gate A completion), which is frequent while the swarm is active and not subject to cron throttling, so zombies are cancelled within minutes; the cron remains as a low-activity backstop. Concurrency switched to `cancel-in-progress: true` to collapse the per-completion burst into a single recent sweep.
