Fixed multiple runners spawned on one host all working the same goal. `relocate_into_agent_worktree` derived the worktree purely from the agent id and `ensure_agent_worktree` reused an existing one, so N runners that shared a single `~/.unsorry/agent-id` piled into the *same* worktree and (for write-access agents) claimed under one identity — no mutual exclusion, so they duplicated each other (worse in claimless fork mode, ADR-068). Each runner now `claim_agent_identity`s a **distinct, lockable identity** at startup: a mkdir-based lock dir holds the owner PID; a live collision forks a random-suffixed id (its own worktree + claim), while a dead owner's lock is reclaimed so the base id is reused across restarts. No `flock`, so it works on macOS. Spawn as many runners as you like — they self-separate without manual `UNSORRY_AGENT_ID` per process.
