Fixed the prove-time claim race that let two agents prove the same goal — the upstream source of the duplicate proof PRs ([ADR-072](docs/adrs/ADR-072-Post-Success-Claim-Recheck.md) / [SPEC-072-A](docs/adrs/specs/SPEC-072-A-Post-Success-Claim-Recheck.md)). Claim files are per-agent (`claims/<goal>.<agent>.aisp`), so two agents claiming the same goal write different paths and the second push is a *clean fast-forward*, not a rejection — and `claim_goal` only rechecked the claim cap on rejection. Both pushes succeeded and both agents proved the goal, creating the sibling `queued/prove/<goal>/*` branches that ADR-064/ADR-071 then only cleaned up downstream. `claim_goal` now runs a **post-success recheck**: after a successful push it re-fetches `origin/claims` and re-applies the per-mode cap (prove 1 / translate 2); if other agents now meet the cap it withdraws the just-pushed claim (via `release_claim`, emits `collision`) and does not prove. Conservative by design — a tight tie makes both racers withdraw and the goal is re-selected next cycle, so there are never two provers on one goal (and never a permanent stall). This stops duplicates at the source rather than sweeping their PRs.
