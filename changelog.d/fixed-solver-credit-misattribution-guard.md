Added a startup credit-integrity guard to `swarm/run.sh` that refuses to launch when `UNSORRY_SOLVER` doesn't match the operator's authenticated GitHub account. `agent.sh:resolve_solver` trusts `UNSORRY_SOLVER` verbatim, so a config that hard-codes someone else's handle silently credits THEM for every proof the machine produces — observed in practice as multiple contributors funnelling leaderboard credit to one handle via a shared `UNSORRY_AGENT_ID=mac-158f` / `UNSORRY_SOLVER` config, undetected until they checked the board. The guard resolves the GitHub login and, on a mismatch, prints the fix and exits non-zero (a scrolling warning in a long-running loop is too easy to miss); deliberate delegation (e.g. proving under an org handle) proceeds with `UNSORRY_SOLVER_OK=1`, and an unresolvable login (offline) warns rather than blocks. The decision is a pure, unit-tested helper (`solver_credit_decision`) covered by `run.sh --self-test`.
