Demand-driven sourcing (`swarm/sourcing.sh --if-pool-empty`) no longer over-sources when a replenishment is already on the way (ADR-084 / SPEC-084-A). The empty-pool gate keyed only on the open-goal count, but an in-flight `chore(sourcing)` PR isn't counted as backlog until it merges (the ADR-067/069 snapshot), so two concurrent sourcers — or one sourcer on its next `UNSORRY_SOURCING_WAIT` tick before its PR merged — both saw an empty pool and both opened a sourcing PR. The gate now also skips when a `chore(sourcing)` PR is already open, mirroring the dispatcher's goal-level dedup (ADR-064/071): a pure `sourcing_gate_decision` (have-work / in-flight / source) plus a best-effort `gh` check made only on an empty pool (core list API, not the rate-limited search). Makes sourcing safe for the decentralised "anyone runs `run.sh`" model; manual `sourcing.sh` is unchanged. New gate logic is unit-tested in the hermetic `--self-test`.
