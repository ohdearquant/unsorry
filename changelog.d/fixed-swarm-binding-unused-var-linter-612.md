The swarm's local proof self-verify (`swarm/agent.sh` `write_binding_module`) now suppresses `linter.unusedVariables` in the regenerated ADR-011 statement-binding obligation, matching Gate A's generator (`tools/gate_a/check_statement_binding.py`). The two had diverged: the swarm built the binding under `lake build UnsorryLibrary --wfail` *without* the `set_option linter.unusedVariables false in` that Gate A emits, so any goal whose hypotheses a correct proof leaves unused (e.g. an inequality true on a wider domain, or a named binder eta-expanded after an implicit one) failed the swarm's self-verify and was needlessly decomposed — even though CI Gate A would accept the proof. The local self-verify is now exactly as strict as the gate, no stricter. Regression self-test `test_binding_module_suppresses_unused_linter` (issue #612).
