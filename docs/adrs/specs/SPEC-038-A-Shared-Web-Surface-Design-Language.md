# SPEC-038-A: Shared Web-Surface Design Language

Implements: [ADR-038](../ADR-038-Shared-Web-Surface-Design-Language.md) · Status: Living · Updated: 2026-06-14

## Scope

The visual design language shared by the public browser surfaces:

- `docs/leaderboard.html` — the contributor leaderboard (issue #270), the page
  that established this language. Hand-maintained; fetches `metrics/leaderboard-ui.json`.
- `docs/proofs-contributors-visualisation.html` — the proof-graph visualiser
  (issue #371). Generated by `tools/visualiser` (`render_html`); the graph model is
  embedded at generation time.

The leaderboard is treated as the **reference**, not as a dependency: the
visualiser re-implements the language in its own generated template. No build-time
coupling between the two files.

## Design tokens

| Token | Value |
|-------|-------|
| Canvas | `#fafbfc` (soft off-white), `-webkit-font-smoothing: antialiased` |
| Card | white, `rounded-3xl`, `shadow-[0_8px_30px_rgb(0,0,0,0.04)]`, `border-slate-100`, centred, max-width |
| Typography | Inter, `system-ui` fallback stack |
| Text / muted | `#334155` / `#64748b` |
| Rank accents | gold `#fbbf24`, silver `#94a3b8`, bronze `#b45309` |
| Progress accent | blue `#e0f2fe` |
| Scrollbar | 6px thumb `#cbd5e1` → `#94a3b8` on hover, transparent track |
| State panel | dashed `#cbd5e1` border, `rounded`, muted text on `#f8fafc` |

Tailwind (CDN) provides layout/utilities; a small inline `<style>` block carries
the tokens above (canvas, badges, state panel, scrollbar) so they are stable
regardless of utility availability. Each surface declares its own
`tailwind.config` `brand` palette with the values above.

## Header pattern

A card header carrying, left-to-right / top-to-bottom on mobile:

1. a large, light "Unsorry" wordmark;
2. an uppercase status/"updated" chip;
3. a cross-link to the sibling surface (visualiser ⇄ leaderboard);
4. a row of summary stat chips (counts), wrapping on narrow viewports.

## Components

- **Badges/chips** — pill, small, with an optional colour swatch. The visualiser
  maps them to the five proof statuses (`proved`/`open`/`blocked`/`flagged`/
  `translated`) reusing the diagram's status fills, so a status badge and a
  leaderboard medal read as the same kit.
- **State panels** — dashed-border empty/error placeholders.
- **Custom scrollbar** — applied to scrollable regions (diagram pane, tables).

## Responsiveness (mobile-first)

- Single-column stack by default; expands to multi-column at `lg`
  (visualiser: diagram + detail panel side-by-side above the table).
- Header wraps (`flex-col` → `md:flex-row`); stat chips wrap.
- Toolbar/controls wrap to multiple rows with adequate tap targets.
- Wide content (the Mermaid diagram, the goals table) scrolls **within** its
  container (`overflow-x-auto`), never forcing page-level horizontal scroll.
- Verified at ~375px (phone), ~768px (tablet), and desktop widths.

## Visualiser specifics (issue #371)

The restyle is presentation-only. It preserves the ADR-032 generation contract:

- the graph model stays embedded as inline JSON (`#graph-data`); the page does not
  fetch at view time;
- the Mermaid forest (`<pre class="mermaid">`, `flowchart LR`), the per-node
  `call showDetail("<goal>")` click wiring, `securityLevel: "loose"`, the
  click-to-detail panel, the zoom controls, and the filterable goals table all
  remain;
- `--write` / `--check` emit and drift-check the `.md` + `.html` + `.svg` set, so the
  post-merge refresh (`proofs-visualisation.yml`) is unchanged.

## Two-surface parity: README preview SVG

Mirroring `docs/leaderboard.svg`, the visualiser also emits `docs/proof-graph.svg`
(`render_svg`, `--svg`): a self-contained README preview card in the shared visual
language — white rounded card, Inter title, a per-status bar breakdown using the
diagram's status fills, plus the decomposition-family count. Unlike the `.md`/`.html`
(whose provenance is git-derived), the SVG is a pure function of the goals' statuses,
so it is deterministic and never drifts on a shallow checkout. The README links it to
the interactive page, alongside the leaderboard preview.

## Tests

`tools/visualiser/tests/test_generate.py` additionally asserts the shared-design
markers in the generated HTML: the viewport meta tag, the Tailwind config `brand`
palette, the Inter font, the responsive grid marker, and the "Unsorry" wordmark —
alongside the existing functional assertions (mermaid pre, `showDetail` callbacks,
embedded-JSON validity, no unreplaced placeholders).
